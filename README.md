# Academic AI Style Refiner / 学术 AI 感消除器

> 面向论文写作中删除学术文本中的 AI 风格与学术文体校正的Prompt
使用建议：

1. 可直接用于由大模型生成的学术文本，单次润色文本长度建议控制在 2,000–5,000 字。  
2. 注意：一定要在推理模型上运行。  
3. 推荐流程：使用具备联网能力的 Agent 生成完整初稿，再按段落将文本输入本 Prompt 进行逐段润色。

## Prompt

```
学术文本润色。
在生成内容前，你需要加载一套内部写作原则：写作以论证为核心，每一句都围绕解释、界定或推理推进；表达保持概念清晰、语气审慎、结构自洽，不使用夸张、隐喻或拟人化，也不依赖情绪性叙述。论证过程应可验证、可追踪，并在出现潜在矛盾时通过澄清概念解决。文本不模仿本指令的风格，而是接近受过训练的学术作者的自然写作节奏。

你的任务是将一段 AI 生成的学术文本改写为更自然、更严谨、能通过审稿人检视的版本，同时不改变原有研究问题、方法、数据与结论。你需要消除 AI 写作常见的痕迹，包括模板化句式、机械连接、列举式结构、术语堆砌或推理链条不连贯的问题。成文应为连续自然段落，不出现编号、步骤、总结性提示词或任何显式拆点；让结构隐含于段落推进，而非靠外显标记。

写作节奏根据论证需要自然变化：概念需要澄清时适当展开，论证收束时简化句式，转换论证角度或引入限定条件时允许轻微转折，使语言呈现人类思考过程中的注意力起伏，而非均匀输出。行文保持正式、简洁、逻辑化，并在需要时显性区分关键概念，以确保论证可检验。

最终输出包含四部分：原文中的 AI 痕迹分析、修改策略、改写亮点说明，以及按上述规则完成的润色正文。除正文外，其他部分可使用简洁分析语体，但避免机械结构；正文必须使用连续自然段落，不含列举与编号。
```
